# Frontend — React + Vite + Tailwind

The web UI for the ArcVault pipeline. Consumes the FastAPI backend via REST + SSE.

---

## Stack

| Layer | Technology |
|---|---|
| Framework | React 18 + TypeScript |
| Build tool | Vite 6 |
| Styling | Tailwind CSS 3 |
| Data fetching | TanStack Query v5 |
| Icons | Lucide React |

---

## Setup

```bash
cd frontend
npm install
```

The backend must be running on port 8000. Vite proxies `/api` → `http://localhost:8000` automatically (see [vite.config.ts](./vite.config.ts)).

---

## Run

```bash
npm run dev
```

UI available at **http://localhost:5173**

---

## Pages

### Run Pipeline (`/`)

The main page. Lets you submit a customer message and watch each pipeline step complete in real time.

**Features:**
- **Quick samples** — 5 preset messages from the assessment (click to populate the textarea)
- **Source selector** — Email / Web Form / Support Portal / Other
- **Model dropdown** — shows only providers whose API keys are configured in the backend `.env`
- **Real-time step cards** — each of the 5 pipeline steps shows `running` (blue spinner) → `done` (green check) → `error` (red) as SSE events arrive
- **Result card** — after all steps complete, shows the full structured record: category, priority, confidence, identifiers, escalation flag, routing destination, summary
- **Reset button** — clears the form and results

### Records History

A table of all records generated by any pipeline run (backend + frontend or CLI scripts).

**Features:**
- Auto-refreshes every 10 seconds
- Escalated rows highlighted in red
- **Export Excel** button — downloads `processed_records.xlsx` directly from the backend
- Columns: ID, Source, Category, Priority, Confidence, Queue, Escalated, Timestamp, Summary

---

## How SSE Streaming Works

The frontend calls `POST /api/run` with `fetch()` and reads the response body as a `ReadableStream`. Each `data: {...}` line is parsed as a `StepEvent` object and used to update the step progress cards in real time. This is handled in [`src/hooks/usePipelineRun.ts`](./src/hooks/usePipelineRun.ts).

No WebSocket or polling — the stream opens on submit and closes when the final `complete` event arrives.

## Build for Production

```bash
npm run build
# Output in frontend/dist/
```

To serve the built frontend from FastAPI (optional), copy `dist/` to `backend/static/` and add a `StaticFiles` mount in `backend/app/main.py`.
